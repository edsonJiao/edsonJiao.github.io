JDBC的学习

发表于 2017-09-08 | 分类于 Java学习笔记

JDBC全称为Java Data Base Connectivity(Java数据库连接)。是连接JAVA与数据库的桥梁。

准备工作

在使用JDBC之前需要在项目目录创建一个lib目录 将mysql的jar包放入，然后build path。


JDBC编程


1.加载驱动：Class.forName(driverClass);
加载MySql驱动:Class.forName(“com.mysql.jdbc.driver”);
加载Oracle驱动:Class.forName(“oracle.jdbc.driver.OracleDriver”);
成功加载后，会将Driver类的实例注册到DriverManager类中。

2.获取数据库连接:
DriverManage.getConnection(url,admin,password) //url为数据库地址本机为”jdbc:mysql://localhost:3306/shop?characterEncoding=UTF-8” admin为数据库账号默认root

3.创建Statement对象
•要执行SQL语句，必须获得java.sql.Statement实例，Statement实例分为以下3种类型：
1、执行静态SQL语句。通常通过Statement实例实现。
2、执行动态SQL语句。通常通过PreparedStatement实例实现。
3、执行数据库存储过程。通常通过CallableStatement实例实现。
具体的实现方式：


       Statement stmt = con.createStatement() ;
       PreparedStatement pstmt = con.prepareStatement(sql) ;
       CallableStatement cstmt =con.prepareCall(“{CALL demoSp(? , ?)}”) ;

4.执行SQL语句
Statement接口提供了三种执行SQL语句的方法：executeQuery 、executeUpdate和execute
1、ResultSet executeQuery(String sqlString)：执行查询数据库的SQL语句，返回一个结果集（ResultSet）对象。
2、int executeUpdate(String sqlString)：用于执行INSERT、UPDATE或DELETE语句以及SQL DDL语句，如：CREATE TABLE和DROP TABLE等
3、execute(sqlString):用于执行返回多个结果集、多个更新计数或二者组合的语句。
具体实现的代码：

    ResultSet rs = stmt.executeQuery(“SELECT * FROM …”) ;
    int rows = stmt.executeUpdate(“INSERT INTO …”) ;
    boolean flag = stmt.execute(String sql) ;

5.处理结果
两种情况：
1、执行更新返回的是本次操作影响到的记录数。
2、执行查询返回的结果是一个ResultSet对象。
• ResultSet包含符合SQL语句中条件的所有行，并且它通过一套get方法提供了对这些
行中数据的访问。
• 使用结果集（ResultSet）对象的访问方法获取数据：

    while(rs.next()){
        String name = rs.getString(“name”) ;
        String pass = rs.getString(1) ; // 此方法比较高效
    }

（列是从左到右编号的，并且从列1开始）

6.关闭JDBC对象
操作完成以后要把所有使用的JDBC对象全都关闭，以释放JDBC资源，关闭顺序和声
明顺序相反：
1、关闭记录集
2、关闭声明
3、关闭连接对象

    if(rs != null){   // 关闭记录集
        try{
            rs.close() ;
        }catch(SQLException e){
            e.printStackTrace() ;
        }
          }
          if(stmt != null){   // 关闭声明
        try{
            stmt.close() ;
        }catch(SQLException e){
            e.printStackTrace() ;
        }
          }
          if(conn != null){  // 关闭连接对象
         try{
            conn.close() ;
         }catch(SQLException e){
            e.printStackTrace() ;
         }
    }


采用MVC架构


View(视图层)Control(控制层)Model(模型层)DB(数据库)
从下往上编写，先写DB层(数据库层)、再写Model层、再写Control层最后写View层
使用JDBC时创建一个dao包用于存放实现类和调用jdbc的方法，创建一个Model包用于存放数据库的类，Action包中存放行为类，控制类


JDBC各种连接方式的对比


1.JDBC+ODBC桥的方式
特点：需要数据库的ODBC驱动、仅适应于微软的系统
2.JDBC+厂商API的形式
特点：厂商API一般由C编写
3.JDBC+厂商Database Connect Server+Database的形式
特点：在JAVA和DATABASE之间架起一台专门用与数据库连接的服务器（一般由数据库厂商提供）
4.JDBC+Database的连接方式
特点：推荐使用，这使得Application和数据库分开，开发者只需要关心内部逻辑的实现而不需要注重数据库连接的具体实现。
